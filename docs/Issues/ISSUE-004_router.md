# ISSUE-004 Echoルーター設定

## 親 Issue
ISSUE-001

## 概要
Echo v4でHTTPサーバーを起動し、全エンドポイントを登録する。JWT認証ミドルウェアを/api/v1グループに適用する。

## 変更ファイル一覧

- `app/src/router/router.go`（編集）
    - **何を**: NewRouter()関数の実装。Echoインスタンス生成、/api/v1グループへのRequireAuth適用、全ハンドラーのルート登録。
    - **なぜ**: HTTPルーティングとミドルウェア適用を一元管理するため
- `app/src/middlewares/auth.go`（編集）
    - **何を**: RequireAuthミドルウェアの実装。JWTトークンを検証してEchoコンテキストにclaimをセットする。
    - **なぜ**: 認証が必要なエンドポイントをグループで保護するため
- `app/src/main.go`（編集）
    - **何を**: router.NewRouter()を呼び出してEchoを起動する処理の追加。
    - **なぜ**: エントリーポイントでルーターを起動する必要があるため

## テスト確認項目

- [ ] 認証なしで/api/v1エンドポイントにアクセスすると401が返ること
- [ ] 有効なJWTトークンでアクセスできること
- [ ] 全エンドポイントが正しいHTTPメソッドで登録されていること
